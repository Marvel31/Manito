# 파파라치 마니또 작업 정리

## 프로젝트 개요

중학교 2학년 반 MT에서 진행할 파파라치 마니또 게임용 웹앱입니다.

참가자는 자신이 뽑은 쪽지의 미션에 맞춰 마니또가 모르게 사진을 찍어 제출하고, 저녁 시간에는 관리자가 제출 사진을 확인하거나 슬라이드 쇼로 보여줄 수 있습니다.

## 기술 구성

- Backend: Node.js, Express
- Upload: Multer
- Frontend: HTML, CSS, Vanilla JavaScript
- 데이터 저장: 서버 파일시스템
- 사진 저장: 서버 파일시스템

## 주요 기능

### 참가자 모드

- 참가자 선택 로그인
- 최초 로그인 시 비밀번호 설정
- 설정한 비밀번호로 재로그인
- 파파라치 사진 1장 업로드
- 사진을 다시 올리면 마지막 사진으로 교체
- 자신의 마니또 예상 선택 및 저장
- 모바일/PC 반응형 화면 지원

### 관리자 모드

- 관리자 비밀번호 로그인
- 참가자 명단 확인
- 참가자 추가, 수정, 삭제
- 참가자 비밀번호 리셋
- 제출 사진과 참가자 예상 마니또 확인
- 1, 2, 3등 선정
- 제출 사진 슬라이드 쇼
- PC 슬라이드 쇼 전체 화면 보기
- 모바일 관리자 메뉴 팝업
- 참가자 추가/수정 팝업

## 관리자 화면 구성

관리자 모드는 메뉴 기반으로 구성했습니다.

- 참가자 명단
- 서브 미션
- 우승자 선정
- Submission 슬라이드 쇼

PC에서는 상단 탭 형태로 보이고, 모바일에서는 제목 오른쪽의 메뉴 버튼을 눌러 팝업 메뉴로 화면을 선택합니다.

## 데이터 저장 위치

기본 로컬 실행 시:

- DB: `data/db.json`
- 업로드 사진: `data/uploads/`

Railway 배포 시:

- `DATA_DIR` 환경변수가 있으면 해당 경로 사용
- 없으면 Railway Volume의 `RAILWAY_VOLUME_MOUNT_PATH` 사용
- 둘 다 없으면 로컬 `data/` 사용

## 실행 방법

```powershell
cd C:\repository\Manito
npm install
npm start
```

접속:

```text
http://localhost:3000
```

관리자 기본 비밀번호:

```text
admin1234
```

관리자 비밀번호 변경 실행:

```powershell
$env:ADMIN_PASSWORD="원하는비밀번호"
npm start
```

## 모바일 로컬 접속

PC와 휴대폰이 같은 Wi-Fi에 연결되어 있어야 합니다.

PC에서 IP 확인:

```powershell
ipconfig
```

휴대폰 브라우저에서 접속:

```text
http://PC의IPv4주소:3000
```

예:

```text
http://192.168.0.23:3000
```

## Railway 배포 설정

Railway에서 GitHub 저장소를 연결한 뒤 다음처럼 설정합니다.

```text
Start Command: npm start
```

환경변수:

```text
ADMIN_PASSWORD=원하는관리자비밀번호
```

사진과 데이터 보존을 위해 Railway 서비스에 Volume을 연결해야 합니다.

## 운영 메모

- 사진은 참가자당 1장만 저장됩니다.
- 같은 참가자가 다시 업로드하면 기존 사진은 삭제되고 새 사진으로 교체됩니다.
- `data/` 폴더는 Git에 포함되지 않습니다.
- Railway 배포 시 Volume을 연결하지 않으면 데이터와 사진이 유지되지 않을 수 있습니다.
- 행사 후에는 사진과 `data/db.json`을 백업하는 것을 권장합니다.
